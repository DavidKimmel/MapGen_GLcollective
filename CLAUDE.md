## Architecture Decisions

- **Rendering:** All maps from OpenStreetMap data at 300 DPI via matplotlib. fig_scale ensures consistent linewidths across all 5 sizes (reference: 16x20).
- **Fulfillment:** Gelato print-on-demand for physical prints. Connected via Ecommerce API (3-step: productUid + print-file + connectionStatus).
- **Digital delivery:** Message buyers with Dropbox link (Etsy's 20 MB limit is too small for large sizes).
- **Listing creation:** Manual via Etsy Listings Manager until API is approved. Cheat sheets (`_listing.txt`, `_variations.txt`) generated by `listing_generator.py`.
- **Variant ordering:** digital (5) → unframed (5) → all framed black (5) → all framed white (5) = 20 per listing.

### Gelato API Connection (3 steps per variant — all required)
1. `PATCH` variant with `productUid` (Gelato product identifier)
2. `POST` print-file with `{"type": "default", "fileUrl": "..."}` (Dropbox URL)
3. `PATCH` variant with `{"connectionStatus": "connected"}` (marks it as connected)

Without step 3, Gelato dashboard shows "not connected" even though files are uploaded.

### Gelato Product UIDs
- Unframed: `flat_{size}-inch-{mm}-mm_170-gsm-65lb-uncoated_4-0_ver`
- Framed black: `framed_poster_mounted_premium_{mm}-{size}_black_wood_w20xt20-mm_plexiglass_{mm}-{size}_200-gsm-80lb-uncoated_4-0_ver`
- Framed white: same pattern with `_white_wood_`

### Listing Workflow (manual, until Etsy API approved)
1. Copy an existing listing in Etsy Listings Manager
2. Update title, description, photos, digital file from `_listing.txt`
3. Sync store in Gelato dashboard
4. Run `python -m etsy.gelato_connect --city "City Name"` to connect variants
5. Publish

## Important Constraints & Patterns

- **Etsy title must match exactly** in Gelato CSV/API — mismatches cause "Product not found"
- **Dropbox tokens are short-lived** (~4 hours) — but shared links persist permanently
- **Gelato CSV import** auto-sets connectionStatus; **API** requires explicit PATCH (step 3)
- **Digital variants** (5 per listing) always show "not connected" in Gelato — this is expected
- **fig_scale** must be applied to all rendering to keep linewidths consistent across sizes
